# Documentation Template

Use this template when updating README files or internal docs. Not every section is required.

## Recommended structure

### What this is
One short paragraph explaining the purpose.

### Why it exists
Explain the problem it solves or the workflow it supports.

### Key files or modules
List the most important files with one-line descriptions.

### How to run or use it
Use concrete commands, inputs, or steps.

### Important assumptions or limitations
Call out anything a teammate could misunderstand.

### Next useful actions
Suggest the logical next step only if it helps the reader act.

## Writing rules

- Prefer bullets over long paragraphs
- Front-load the most important actions
- Use exact file names and commands when possible
- Be explicit about what is intentionally incomplete in demo code
- Keep the tone practical and teammate-friendly
